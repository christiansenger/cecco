CECCO: Prime Fields

CECCO supports finite extension fields, their arithmetic, and their properties. The ingredients for a finite extension field FQ with Q=q^m elements are a finite base field Fq with q elements and an irreducible polynomial P over Fq of degree m. Since CECCO realizes finite extension fields based on lookup tables (that are, depending on the configuration, compiler-generated), both Fq and P must be available at compile time.

Start with any finite field Fq. If q is a prime, the field can be constructed right away. If Fq itself is a finite extension field, we must successively calculate irreducible polynomials until a prime field is reached.

In case Q=2=2^1, we have q=2 and the base field is the prime field F2. It can be constructed right away.

code.cpp Lines 8–8 
    using F = Fp<2>;

In case Q=4=2^2, we still have q=2 and the base field is the prime field F2 (which can be constructed right away). We require an irreducible (over F2) polynomial of degree 2. This results in the finite field with four elements F4.

code.cpp Lines 9–9 
    // using F = Ext<Fp<2>, {1, 1, 1}>;                                                     // F4

In case Q=8=2^3, we still have q=2 and the base field F2 (which can be constructed right away). We require an irreducible (over F2) polynomial of degree 3. This results in the finite field with eight elements F8.

code.cpp Lines 10–10 
    // using F = Ext<Fp<2>, {1, 0, 1, 1}>;                                                  // F8

In case Q=16=2^4, we still have q=2 and the base field F2 (which can be constructed right away). We require an irreducible (over F2) polynomial of degree 4. This results in the finite field with 16 elements F16. But we can also write Q=16=4^2, so an alternative way of constructing F16 is based on F4 (which is an extension field by itself and an irreducible polynomial (over F4) of degree 2. This also results in F16 (up to isomorphism):

Here, F16 is first constructed as Ext, {1, 0, 0, 1, 1}>, then as Ext, {1, 1, 1}>, {2, 2, 1}>, and finally both constructions are merged into a family of isomorphic fields with 16 elements that can be used as any finite field.

code.cpp Lines 11–11 
    // using F = Iso<Ext<Fp<2>, {1, 0, 0, 1, 1}>, Ext<Ext<Fp<2>, {1, 1, 1}>, {2, 2, 1}> >;  // F16

Such constructions can also be realized for other primes besides 2, for example:

code.cpp Lines 13–13 
    // using F = Ext<Fp<3>, {1, 2, 0, 1}>;                                                  // F27

In the examples above, we did not explain how we obtained the coefficients of the irreducible polynomials. They can in fact be found with the help of CECCO by either trying to find a random irreducible (over Fq) polynomial of the right degree as in

code.cpp Lines 15–18 
    constexpr size_t m = 6;
    auto P = find_irreducible<F>(m);
    std::cout << "Monic irreducible polynomial: " << P << std::endl;
    std::cout << "Coefficient vector (to be used as template parameter of Ext<>): " << Vector(P) << std::endl;

Monic irreducible polynomial: 1 + x^2 + x^6
Coefficient vector (to be used as template parameter of Ext<>): ( 1, 0, 1, 0, 0, 0, 1 )

or, alternatively, by using one of the well-known Conway polynomials that are hard-coded into the library:

code.cpp Lines 20–22 
    auto Q = ConwayPolynomial<p, m>();
    std::cout << "Conway polynomial: " << Q << std::endl;
    std::cout << "Coefficient vector (to be used as template parameter of Ext<>): " << Vector(Q) << std::endl;

Conway polynomial: 1 + x + x^3 + x^4 + x^6
Coefficient vector (to be used as template parameter of Ext<>): ( 1, 1, 0, 1, 1, 0, 1 )

Note that both polynomials and coefficient vectors are consistently written with they least significiant coefficient/component on the left.

The coefficient vectors that are obtained in this way are supposed to be hard-coded as template parameters of Ext<>, same as in the introductory examples F2, F4, F16, and F27!

A complete, compilable demo is shown below:

code.cpp 
#include <iostream>

#define CECCO_ERASURE_SUPPORT
#include "cecco.hpp"
using namespace CECCO;

int main(void) {
    using F = Fp<2>;
    // using F = Ext<Fp<2>, {1, 1, 1}>;                                                     // F4
    // using F = Ext<Fp<2>, {1, 0, 1, 1}>;                                                  // F8
    // using F = Iso<Ext<Fp<2>, {1, 0, 0, 1, 1}>, Ext<Ext<Fp<2>, {1, 1, 1}>, {2, 2, 1}> >;  // F16

    // using F = Ext<Fp<3>, {1, 2, 0, 1}>;                                                  // F27

    constexpr size_t m = 6;
    auto P = find_irreducible<F>(m);
    std::cout << "Monic irreducible polynomial: " << P << std::endl;
    std::cout << "Coefficient vector (to be used as template parameter of Ext<>): " << Vector(P) << std::endl;

    auto Q = ConwayPolynomial<p, m>();
    std::cout << "Conway polynomial: " << Q << std::endl;
    std::cout << "Coefficient vector (to be used as template parameter of Ext<>): " << Vector(Q) << std::endl;

    return 0;
}
